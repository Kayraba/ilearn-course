# Incident Response - Pilot Draft

Use this if there is a suspected privacy, security, safeguarding or availability
incident.

## Contacts

- Product/operator lead: `[name/contact]`
- Client lead: `[name/contact]`
- Data protection contact: `[name/contact]`
- Safeguarding lead: `[name/contact]`
- Hosting provider: `[provider/support link]`

## First hour

1. Record what happened, when, who noticed it and what systems are affected.
2. Preserve logs and screenshots.
3. Stop further exposure if safe to do so.
4. Notify the responsible operator/client contact.
5. Decide whether learner data or safeguarding information is involved.

## Investigation

- identify affected learner/staff records
- check audit logs
- check hosting/deployment logs
- confirm whether credentials or secrets are exposed
- document actions taken

## Communication

Do not make unsupported claims. Use clear facts, known impact, actions taken and
next update time.

## Post-incident

- fix root cause
- rotate affected credentials/secrets
- review safeguards
- update this document
- record lessons learned
