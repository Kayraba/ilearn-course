# iLearn voiceover and tutor-script upgrade

## What changed

This build no longer relies only on the browser's built-in speech voice. Browser speech can sound robotic, cut out on longer lines, or change between devices. The app now supports real recorded MP3 narration for every lesson beat.

The recommended free voice workflow is **Microsoft Edge Neural TTS** through the `edge-tts` tool.

Recommended voice settings:

```bash
EDGE_VOICE=en-GB-SoniaNeural
EDGE_RATE=-8%
EDGE_PITCH=+0Hz
```

These settings create a calm British tutor voice that is easier for supported learners to follow.

## Generate the full narrated audio

From this folder, run:

```bash
npm run voiceover:free
```

This will:

1. Install `edge-tts` if needed.
2. Read the actual lesson timeline from `lesson-video.js`.
3. Generate one MP3 for each spoken beat.
4. Save the audio as `audio/lesson-01-01.mp3`, `audio/lesson-01-02.mp3`, etc.
5. Create `audio/voiceover-manifest.json`.
6. The app will automatically use the generated MP3s when the manifest exists.

To generate only one lesson while testing:

```bash
LESSON=1 npm run voiceover:free
```

To regenerate audio even if MP3s already exist:

```bash
FORCE=1 npm run voiceover:free
```

On Windows PowerShell:

```powershell
$env:LESSON="1"; npm run voiceover:free
```

## Tutor-script improvements

The script was adjusted to sound more like a real tutor:

- Warmer opening lines.
- One instruction at a time.
- Less robotic phrasing.
- More reassurance after mistakes.
- More natural transitions into practice.
- Clearer “watch, then try” teaching flow.

The new teaching voice follows this pattern:

> “I will show you first. Then you will try it yourself. Go slowly. Mistakes are part of learning.”

## Caption fixes

Captions now:

- Stay above the animation layer.
- Do not sit behind the Word/Excel mockup.
- Are limited in height so long lines do not cover the whole video.
- Use stronger contrast.
- Work better on mobile/tablet.
- Respect reduced-motion settings.

## Existing `audio/lesson-01.mp3` files

The files named `audio/lesson-01.mp3`, `audio/lesson-02.mp3`, etc. are older short lesson intro files. They do not cover the full detailed lesson timeline.

For the smoothest result, use the new per-beat files generated by:

```bash
npm run voiceover:free
```

The app looks for `audio/voiceover-manifest.json`; when it exists, it uses the proper per-beat MP3 narration.

## Activity-aligned tutor scripts

The script now includes an activity-guide section for every lesson. This means the tutor explains how the activities below the animation connect to the lesson.

Each lesson now covers:

- What the learner is learning
- A step-by-step demonstration
- What the warm-up activity will do
- What the quick check questions are for
- What real-computer practice is expected
- What typed/reflection task may appear
- A common mistake to avoid
- A calm closing before the learner starts the activities

The scripts are intentionally calm and slightly longer than a rushed 2-minute video, because the target learners need enough time to understand what is happening.
